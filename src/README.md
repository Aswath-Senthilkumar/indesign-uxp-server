# src — MCP Server

The Model Context Protocol (MCP) server. Exposes ~135 InDesign tools to AI assistants (Claude, Cursor, etc.) via stdio transport. This layer owns only **tool definitions and handler dispatch** — all runtime state lives in `sessionManager.js` and all InDesign execution routes through `scriptExecutor.js` → bridge → plugin.

## Session manager

`sessionManager.js` maintains lightweight per-session state between tool calls:

- **Document dimensions** — width + height in points; set when a document is opened/created
- **Active page** — current page index; updated by navigation tools
- **Bounds validation** — rejects coordinates that would place items outside the page
- **Prototype pollution protection** — `stripDangerousKeys()` sanitises all incoming objects

This is what allows an LLM to say "place a text frame in the top-right corner" and receive a valid bounds rect without supplying raw points.

## Code execution pattern

Every handler eventually calls `executeViaUXP(code)` from `scriptExecutor.js`. The code string is a self-contained async IIFE evaluated inside InDesign:

```js
// generated by a handler:
const code = `
    const { SaveOptions, ExportFormat } = require('indesign');
    let result;
    try {
        // ... handler-specific logic ...
        result = { ok: true, data: { ... } };
    } catch(e) {
        result = { ok: false, error: e.message };
    }
    return result;
`;
```

The bridge relays it to the UXP plugin, which runs it as:
```js
new Function('app', `return (async () => { ${code} })()`)(app)
```

## Startup

```bash
# from repo root
npm install
npm start
```

Requires bridge running on `:3000`/`:3001` and the UXP plugin panel open in InDesign. The server exits with a clear error if the bridge is unreachable at startup.

```json
{
  "mcpServers": {
    "indesign": {
      "command": "node",
      "args": ["/path/to/indesign-uxp-server/src/index.js"]
    }
  }
}
```
